**幻灯片3：什么是PE文件？**
- PE = Portable Executable（可移植可执行）
- Windows平台的标准可执行文件格式
- 常见扩展名：
  - .exe（可执行程序）
  - .dll（动态链接库）
  - .sys（系统驱动）
  - .ocx（ActiveX控件）
  - .scr（屏幕保护）

**幻灯片4：PE文件结构总览**
```
┌─────────────────────────┐
│  DOS头 (DOS Header)      │
├─────────────────────────┤
│  DOS存根程序 (DOS Stub)  │
├─────────────────────────┤
│  NT头 (NT Header)        │
├─────────────────────────┤
│  节表 (Section Headers)  │
├─────────────────────────┤
│  节数据 (Section Data)   │
│  ├─ .text (代码节)       │
│  ├─ .data (数据节)       │
│  └─ ...                 │
└─────────────────────────┘
```

**幻灯片5：关键概念**
- **文件偏移**：数据在文件中的实际位置
- **虚拟地址（VA）**：程序加载到内存后的地址
- **相对虚拟地址（RVA）**：相对于ImageBase的偏移
- **RAW**：节在文件中的位置

**幻灯片6：VA、RVA、RAW的关系**
- RVA = VA - ImageBase
- 文件偏移 = RVA - 节的VirtualAddress + 节的PointerToRawData
- （需要注意对齐问题）

**幻灯片5：INT vs IAT**
- INT（Import Name Table）：存储函数名称或序号

- 加载前：INT和IAT指向相同数据
- 加载后：IAT被填充为实际函数地址

**幻灯片6：IMAGE_THUNK_DATA**

- 最高位为1：按序号导入

- IMAGE_BASE_RELOCATION结构体
- 结构组成：
  - VirtualAddress：重定位块的起始RVA
  - SizeOfBlock：块大小
  - TypeOffset数组：类型和偏移数组

**幻灯片5：重定位类型**
- 0：绝对（忽略）
- 1：高16位（16位）
- 2：低16位（16位）
- 3：高16位加低16位（32位）← 最常用

**幻灯片6：重定位计算**
- Delta = 实际基址 - ImageBase
- 需要重定位的地址 += Delta
- 图解重定位过程

**幻灯片7：重定位修复步骤**
1. 计算Delta
2. 遍历每个重定位块
3. 遍历块中的每个重定位项
4. 根据类型修复地址
5. 继续下一项/下一块
